Sass

Sass 是 CSS 的預處理器(preprocessor)，目的在撰寫 css 時減少反覆重工的動作以及更有效率的樣式結構維護，Sass 必須經過編譯 (compiler) 或原生的 CSS 才能使用，也就是說瀏覽器不懂 Sass，需經過編譯才能進行閱讀。

CSS 不是程式語言，沒有流程控制、函式這些功能，Sass 除了可以使用嵌套規則 (nested rules)、繼承 (extends)、混合 (mixins) 這些功能，還可以使用變數 (variables)、函式 (Function)、流程控制 (Flow Control) 等等，所以也可以稱為 CSS 的程式語言。

Sass 分為兩種 Sass 與 Scss，都稱為 Sass。不同點如下：

1. 變數 (Variables)

變數使用時機，當一個值被重複使用兩次以上時，就可考慮使用

變數可以統一定義網站規範，只須修該變數值，不需在 CSS 原生檔一一修改數值

底線 "_" 和連接線 "-" 的使用可以互換。定義變數名稱是 $bg_black，但可以透過 $bg-black 提取

2. 嵌套 (Nested)

。嵌套雖然很好使用，不表示所有層級都該寫入嵌套裡面，可考慮使用最少的選擇器選擇到位置

。過度嵌套編譯後選擇器過長，會影響到效能的執行，也會造成 CSS 檔案容量過大

。以區塊嵌套有個優點，partial 的概念讓結構區塊模組化，專案維護時只需針對錯誤找到模組位置修改

3. 繼承 (Extend)

。繼承可以使用 css 選擇器或 % 開始命名

。使用 css 選擇器，會被編譯到 css，而 % 佔位符不會被編譯到 css

。使用 @extend 可以養成習慣，統一將其放在樣式的最前面

。相同的繼承樣式在編譯後會被使用群組選擇器合併再一起

4. 計算

可以做加、減、乘、除、取餘數的計算

。除法比較特別，遇到除法計算，養成用括號的習慣

。Sass 的計算與 Calc() 不同點在於預處理前已經計算好結果，不會造成瀏覽器的負擔，Calc() 是在文件正被瀏覽器解讀時進行，會消耗瀏覽器資源

5. 函式 (function)

@funtion 與 @mixin 有點像，都是在做運算處理，兩個差異在於 @function 必須使用 @return 來返回結果，而且輸出的是一個值，並非是一段 css 樣式，@mixin 則可以使用 @include 產生一段樣式

6. 混合 (Mixins)

@mixin 就像 @function 可下參數，大括號裡面可以存放變數、自訂樣式、函式，是一種重複使用的樣式加工概念，產生一組可重複使用的樣式，並使用 @include 放置在選擇器內容中

使用 mixin 建立按鈕

mixins 多參數 + for：建立多個不同顏色的盒子

1. 多參數會以 list 型態帶入

2. @for 有兩種:

@for $i from (start number) through (end number){...} 會包括 end number

@for $i from (start number) to (end number){...} 會排除 end number

8. 資料型態

  • 1. number(數字)：數字有分有單位或無單位，1,2,13,10px
  • 2. string(字串)：有引號字串，"foo"、"bar"、無引號字串 baz
  • 3. color(顏色)：blue,#000000,rgba(0,0,0,0.5)
  • 4. boolean(布林值)：true false
  • 5. null(空值):null
  • 6. list(像陣列)：用空格或是逗號作為分隔，1.5rem 1rem 0 2rem 或 "Helvetica,Arial,sans-serif
  • 7. map(物件)：類似 js 的物件

9. Partials (拆分再組織結構的概念)

1. 以底線來為 Partials 檔案命名，Sass 會自行判斷底線開頭為 Partial 檔，有底線的 Partial 檔不會產生 CSS 檔

2. 載入 Partials 的方式為：@import 'variables'，Partials 檔的底線和副檔名可以省略

3. 許多 Partials 檔，例如：mixins 等等，會被用一支主要的 sass 檔來載入他們
